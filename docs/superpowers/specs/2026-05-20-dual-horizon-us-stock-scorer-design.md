# 双周期美股买入时机评分器设计

日期：2026-05-20

## 目标

构建一个个人使用的微信小程序原型。用户输入一个美股 ticker，系统同时给出：

- 中长期判断：未来 3-12 个月是否值得持有或分批建仓。
- 短期判断：未来 1-2 周，即 5-10 个交易日内，当前是否适合入场。
- 当前行动建议：可分批买入、可小仓试探、观察等待、只适合短线、回避。

系统定位是个人投资研究工具，不直接承诺收益，不做保证性买卖建议。所有结论都需要显示主要依据、触发条件和风险失效条件。

## 非目标

- 不预测明天涨跌。
- 不做日内交易策略。
- 不直接管理真实资金账户。
- 不提供自动下单。
- 不把单一综合分包装成确定性买卖信号。
- 第一版不追求覆盖所有美股，先覆盖用户关注列表和流动性较好的大型/中型股票。

## 核心原则

1. 中长期和短期分开评分，最后再合成行动建议。
2. 评分必须可解释，每个分项都能展示正面和负面证据。
3. 回测评价不用“准确率”作为唯一指标，而是看分层收益、回撤、换手率、交易成本后收益和样本外表现。
4. 数据必须尽量避免幸存者偏差、复权错误、财报发布日期错误和未来函数。
5. 第一版优先稳定、透明、可手工复核，而不是追求复杂机器学习。

## 数据来源

### 原型阶段

适合个人原型和每日展示：

- Polygon / Massive：日线、分钟线、公司财报、部分 ratios、公司事件。适合生产 API。
- Financial Modeling Prep：财报、比率、分析师预期、企业日历较方便。适合快速原型。
- Alpha Vantage：复权日线、财报、分红拆股等，适合低成本备选。
- SEC EDGAR API：官方财报源，用于关键字段校验，尤其是 10-K、10-Q 和 filing date。
- Kenneth French Data Library：用于 Fama-French 因子归因和市场基准验证。

### 严肃回测阶段

如果要做 5-10 年以上的可信回测，建议使用：

- Nasdaq Data Link / Sharadar：覆盖 active 和 delisted 股票，有价格、基本面、公司行动和点时维度，适合个人/小团队量化研究。
- CRSP + Compustat：机构级数据，质量最高但成本高，适合后续升级。

### 数据质量要求

最低要求：

- 价格使用 split/dividend adjusted close。
- 财报字段使用 filing date 或 accepted date，不能用 period end date 直接参与历史回测。
- 股票池需要过滤 ETF、基金、优先股、权证、低价股和极低流动性股票。
- 回测阶段必须考虑退市股票，否则会严重高估收益。
- 指标计算需要记录数据更新时间和来源，方便排查异常。

## 股票池

第一版默认过滤：

- 普通股为主，排除 ETF、ETN、ADR、SPAC、权证、优先股。
- 股价大于 5 美元。
- 20 日平均成交额大于 2,000 万美元。
- 市值大于 5 亿美元。
- 最近一个财报周期内财务数据可用。

这些阈值保留为配置项。个人关注列表中的股票可以允许手工加入，但界面要标注“数据不足”或“流动性不足”。

## 中长期评分模型

周期：3-12 个月。

输出：0-100 分，外加五个分项解释。

建议权重：

| 因子 | 权重 | 核心指标 |
|---|---:|---|
| 质量/盈利 | 25% | ROIC、ROE、毛利率、经营利润率、自由现金流质量、应计利润 |
| 估值 | 20% | Forward PE、EV/EBITDA、FCF Yield、Earnings Yield、行业内估值分位 |
| 成长与预期 | 20% | 收入增速、EPS 增速、利润率趋势、分析师预期修正，如果无预期数据则使用历史增长替代 |
| 投资纪律/财务稳健 | 15% | 资产增长率、资本开支强度、净负债率、利息覆盖、股本稀释 |
| 中期动量与风险 | 20% | 6-12 个月相对强弱、回撤、Beta、波动率、行业相对表现 |

标准化方式：

- 对每个指标做 winsorize，默认截断 1% 和 99% 极端值。
- 优先做行业内 percentile rank，再合成为分项分数。
- 对明显越低越好的指标反向处理，例如估值倍数、负债率、波动率。
- 若数据缺失，分项降权，不用 0 分硬惩罚；但缺失过多时给出“数据不足”结论。

中长期分档：

| 分数 | 判断 |
|---:|---|
| 80-100 | 强，适合重点关注或分批建仓 |
| 65-79 | 偏强，具备中长期配置价值 |
| 50-64 | 中性，需要等待更好价格或基本面确认 |
| 35-49 | 偏弱，不适合作为中长期仓位 |
| 0-34 | 弱，回避中长期持有 |

## 短期评分模型

周期：1-2 周，约 5-10 个交易日。

输出：0-100 分，外加短期状态标签。

建议权重：

| 模块 | 权重 | 核心指标 |
|---|---:|---|
| 趋势状态 | 30% | 价格相对 20/50/200 日均线，均线斜率，短趋势是否向上 |
| 回调质量 | 25% | 是否健康回踩 20 日均线或前高支撑，成交量是否缩量，是否出现放量破位 |
| 短期动量 | 20% | 5 日、10 日、20 日收益相对 SPY/QQQ/行业 ETF 的强弱 |
| 过热风险 | 15% | RSI、价格偏离 20 日均线、ATR 扩张、连续跳空或连续大阳线 |
| 事件风险 | 10% | 财报日、FOMC、CPI、重大公告、诉讼/监管、异常波动 |

短期标签：

| 标签 | 解释 |
|---|---|
| 可入场 | 趋势向上，回调或突破质量较好，未明显过热 |
| 可小仓试探 | 条件基本满足，但确认不足或波动偏高 |
| 等回调 | 中长期或趋势不错，但短期价格过热 |
| 等突破 | 横盘或压力位附近，方向未确认 |
| 回避 | 跌破关键趋势、放量下跌、事件风险过高 |

短期分数只是辅助。若事件风险过高，例如财报前 1-2 个交易日，系统可以用风险标签覆盖高分，输出“等待事件落地”。

## 双周期决策矩阵

| 中长期 | 短期 | 当前行动 |
|---|---|---|
| 强 | 强 | 可分批买入 |
| 强 | 中性/等待 | 好公司，等待更好买点 |
| 强 | 过热 | 不追高，设置回调提醒 |
| 强 | 弱/破位 | 长期关注，但等待企稳 |
| 中性 | 强 | 可小仓观察，不做重仓 |
| 中性 | 弱 | 观察等待 |
| 弱 | 强 | 只适合短线，不进入长期仓位 |
| 弱 | 弱 | 回避 |

行动建议必须同时给出：

- 建议动作：分批、小仓、等待、回避。
- 触发条件：例如回踩 20 日均线企稳、放量突破前高、财报后指引改善。
- 失效条件：例如跌破 50 日均线、跌破最近 swing low、基本面预期下修。
- 风险说明：例如估值过高、财报临近、波动率异常、行业整体走弱。

## 示例输出

```text
Ticker: MSFT

中长期评分：82/100
判断：优质资产，盈利质量和现金流强，估值略高但仍可接受。

短期评分：58/100
判断：趋势仍强，但短线偏高，追入性价比一般。

当前行动：观察等待
触发条件：回踩 20 日均线附近企稳，或放量突破前高。
失效条件：跌破 50 日均线且相对 QQQ 转弱。
风险提示：短期涨幅较大，若财报或宏观利率预期变化，波动可能放大。
```

## 回测与验证

回测对象：

- 标普 500 成分股历史池。
- 纳斯达克 100 成分股历史池。
- 用户自选池。
- 若使用 Sharadar 或 CRSP，应包含退市股票。

中长期验证：

- 月频调仓。
- 持有期 3、6、12 个月分别统计。
- 比较 Top 10%、Top 20%、Middle、Bottom 20% 分层表现。
- 关注年化收益、超额收益、最大回撤、Sharpe、信息比率、换手率、交易成本后收益。

短期验证：

- 每日收盘后生成信号，下一交易日开盘或 VWAP 入场模拟。
- 持有期 5、10 个交易日。
- 统计平均收益、中位数收益、胜率、盈亏比、最大不利变动、最大有利变动。
- 分开统计大盘上行、震荡、下行环境。

防止过拟合：

- 使用 walk-forward 测试。
- 先固定指标和权重，再做样本外验证。
- 不用未来财报数据。
- 不因为个别股票表现反复调参。
- 单独记录交易成本和滑点。

## 系统架构

第一版推荐架构：

```text
数据源 API
  -> 定时 ETL
  -> 原始数据表
  -> 指标计算任务
  -> 股票评分表
  -> 小程序后端 API
  -> 微信小程序前端
```

后端职责：

- 管理第三方 API key。
- 拉取和缓存价格、财报、公司事件。
- 计算指标和评分。
- 保存历史评分，支持回测。
- 暴露 ticker 查询接口。

小程序前端职责：

- 输入 ticker。
- 展示中长期评分、短期评分和当前行动建议。
- 展示因子解释、触发条件、失效条件。
- 支持自选列表和提醒。

## 第一版范围

MVP 功能：

- 输入 ticker 查询。
- 显示公司基础信息和最新价格。
- 显示中长期五因子评分。
- 显示 1-2 周短期买点评分。
- 显示双周期行动建议。
- 显示关键证据和风险标签。
- 支持 20-50 只自选股每日更新。

暂缓功能：

- 自动交易。
- 组合优化。
- 盘中实时信号。
- 新闻情绪模型。
- 复杂机器学习模型。
- 微信支付和多用户订阅。

## 实施顺序建议

1. 先做 Python 本地研究脚本，验证 20-50 只股票的评分输出。
2. 接入一个低成本数据源，补充 SEC EDGAR 校验关键财报字段。
3. 保存每日评分快照。
4. 做 5 年以上回测，如果数据源不足，先只做“近似验证”，并标注局限。
5. 搭建后端 API。
6. 做微信小程序页面。
7. 再考虑提醒、自选列表和组合层面的风险控制。

## 主要风险

- 免费或廉价数据源可能有缺失、延迟、复权不一致。
- 没有退市股票时，回测会偏乐观。
- 短期 1-2 周信号对市场环境非常敏感，不能脱离 SPY/QQQ 和行业 ETF 判断。
- 财报、监管、宏观事件可能让技术指标失效。
- 单只股票风险无法只靠因子分数解决，需要仓位和失效条件配合。
